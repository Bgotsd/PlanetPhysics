## 汶川地震数据

UTC时间：2008-05-12 06:28:01

震中位置：**31.021**, **103.367**

30-90度的地震台站的含义是：相隔圆心角为该度数的地震台站，而非方位角

因此对应miniradius=30 maxradius=90

因为地球是一个球体，用圆心角（度）来衡量距离比公里更直观，尤其是在研究地震波穿透地球内部的路径时。

* $1^\circ \approx 111 \text{ km}$
* $30^\circ \approx 3,330 \text{ km}$
* $90^\circ \approx 10,000 \text{ km}$

考虑地震波传播的速度，至少获得震后30min内数据

# URL 结构

顺序不重要，但**结构**必须固定：
`协议://域名/路径/版本/方法?参数1=值1&参数2=值2&参数3=值3`

* **路径部分** （`fdsnws/station/1/query`）顺序 **不能乱** 。
* **参数部分** （问号 `?` 之后的部分）顺序 **可以乱** 。

服务器在接收到你的请求时，会自动寻找问号 `?` 后面的键值对（Key-Value Pairs）。只要你的参数名（如 `net`, `sta`, `minlatitude` 等）拼写正确，且每个参数之间用 `&` 符号连接，服务器就能识别它们。

# Bulk格式

Bulk 格式的每一行代表一个具体的数据提取指令，字段之间用**空格**分隔：

> **`网络代码` `台站代码` `位置标识符` `通道代码` `开始时间` `结束时间`**

```bash
IU ANMO 00 BHZ 2008-05-12T06:28:01 2008-05-12T07:28:01
II KIV  * BH* 2008-05-12T06:30:00 2008-05-12T07:00:00
IC WMQ  -- LHZ 2008-05-12T06:28:01 2008-05-12T08:28:01
```

* **`*` (通配符)** ：代表匹配该字段下的所有内容（如 `BH*` 匹配所有垂向、水平分量）。
* **`--` (空白位)** ：专门用于表示没有 Location ID 的台站。
* **时间精度** ：通常遵循 ISO8601 格式（`YYYY-MM-DDTHH:MM:SS`）。
