# URL 结构

## URL的基本格式

`协议://域名/路径/版本/方法?参数1=值1&参数2=值2&参数3=值3`

主要包括两个部分：

* **路径部分** （`fdsnws/station/1/query`）顺序 **不能乱** 。
* **参数部分** （问号 `?` 之后的部分）顺序 **可以乱** 。

服务器在接收到你的请求时，会自动寻找问号 `?` 后面的键值对（Key-Value Pairs）。只要你的参数名（如 `net`, `sta`, `minlatitude` 等）拼写正确，且每个参数之间用 `&` 符号连接，服务器就能识别它们。

# 地震学数据

## 波形数据

### Bulk格式

Bulk 格式的每一行代表一个具体的数据提取指令，字段之间用**空格**分隔：

> **`网络代码` `台站代码` `位置标识符` `通道代码` `开始时间` `结束时间`**

```bash
IU ANMO 00 BHZ 2008-05-12T06:28:01 2008-05-12T07:28:01
II KIV  * BH* 2008-05-12T06:30:00 2008-05-12T07:00:00
IC WMQ  -- LHZ 2008-05-12T06:28:01 2008-05-12T08:28:01
```

* **`*` (通配符)** ：代表匹配该字段下的所有内容（如 `BH*` 匹配所有垂向、水平分量）。
* **`--` (空白位)** ：专门用于表示没有 Location ID 的台站。
* **时间精度** ：通常遵循 ISO8601 格式（`YYYY-MM-DDTHH:MM:SS`）。

## Trace

表示一个台站某个分量的数据

* `st[0]`：垂直分量 ( **Z** )
* `st[1]`：北向分量 ( **N** )
* `st[2]`：东向分量 ( **E** )

虚拟台网：

* **`_GSN`** ：包含所有的全球骨干网（IU, II, IC, CU 等）。**这是搜索大地震波形最方便的选项。**
* **`_REALTIME`** ：仅包含目前有实时数据回传的台站。
* **`_USALL`** ：包含美国境内所有的台网。

### `cha`

 `?HZ` 含义如下：

* **`?`** ：匹配任意一个字母（通常代表采样率，如 `B` 代表 Broad Band，`H` 代表 High Speed）。
* **`H`** ：代表高增益（High Gain）。
* **`Z`** ：代表 **垂直方向** (Vertical)。

**推荐配置：**

1. **只想看垂直方向：** 保持 `?HZ` 或 `BHZ`。
2. **想下载三分量（垂直+东西+南北）：** 改为 **`BH?`** 或 **`HH?`** 或  **`?H?`** 。
3. **最保险的写法：** 如果不知道台站有哪些通道，直接填  **`*`** ，但这样下载的数据量会很大。

第一位：采样率 (Band Code)

| **代码** | **含义**                  | **特点**                                       |
| -------------- | ------------------------------- | ---------------------------------------------------- |
| **H**    | **High Gain Seismometer** | **最常用** 。高增益地震计，灵敏度高。          |
| **L**    | **Low Gain**              | 低增益，通常用于强震记录（防止大震时信号“打顶”）。 |
| **N**    | **Accelerometer**         | 加速度计，专门记录强地动。                           |

第二位：仪器类型 (Instrument Code)

| **代码** | **含义**                  | **特点**                                       |
| -------------- | ------------------------------- | ---------------------------------------------------- |
| **H**    | **High Gain Seismometer** | **最常用** 。高增益地震计，灵敏度高。          |
| **L**    | **Low Gain**              | 低增益，通常用于强震记录（防止大震时信号“打顶”）。 |
| **N**    | **Accelerometer**         | 加速度计，专门记录强地动。                           |

第三位：方向(Orientation Code

| **代码**      | **含义**        | **描述**                                                        |
| ------------------- | --------------------- | --------------------------------------------------------------------- |
| **Z**         | **Vertical**    | 垂直方向（上下）。                                                    |
| **N**         | **North-South** | 地理南北方向。                                                        |
| **E**         | **East-West**   | 地理东西方向                                                          |
| **1 / 2 / 3** | **Orthogonal**  | 某些台站不按南北摆放，而是三个互相垂直的轴，通常 1=北, 2=东, 3=垂直。 |
