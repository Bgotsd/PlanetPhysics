## Obspy的局限性

Obspy作为一个大型的地震学python包，为了运行速度和格式上的统一，会修改原有的工具包的一些功能以及**版本**。例如在本次作业复现震相图时，基本的流程应该是构建速度模型（.nd文件）->编译为二进制文件->绘制图像。但Obspy内置的taup就一直因为我构建的火星无内核模型（Mars_With_No_Core,MNC)没有正常的壳层结构而一直编译失败导致无法绘制图像，我一度以为这是taup库的缺陷。直到我看到官方文件展示如何构建Europa的速度模型的时候，我就知道这几把Obspy是真的傻逼。

## *`<u>`**原因** `</u>`*

Obspy为了稳定性，现有的Taup***`<u>`依旧是2.0版本 `</u>`***，**仅仅支持格式mantle, outer-core,inner-core**

ps：也不排除是我安装的Obspy太落后了，同志们可以检查下Obspy的更新

> The second type of line within the .nd format specifies one of the major internal boundaries. The original format **was limited to *mantle*, *outer-core* and *inner-core***, but in version 3.0 this was **expanded to include more crustal boundaries and synonyms**.

而在新更新的Taup 3.0版本中，极大拓展了界面的限制（Obspy你不更新我们学行星科学的不完犊子了）

| Labels              | Description                 |
| :------------------ | :-------------------------- |
| crust               | top of crust                |
| mantle, moho        | moho, crust-mantle boundary |
| outer-core, cmb     | mantle-core boundary        |
| inner-core, icocb   | inner outer core boundary   |
| ice                 | top of ice layer            |
| ice-ocean           | ice above ocean boundary    |
| ice-crust           | ice above crust boundary    |
| ocean               | top of ocean layer          |
| seabed, ocean-crust | ocean above crust boundary  |

## 速度模型构建

官方支持的速度模型主要有两种，一种是 `.tvel`，一种是 `.nd`。一般而言都使用后者

注意每行格式满足以下顺序，一般只需要给出一些重要深度的参数，其他的深度Taup会自动线形插值

| Depth | Kilometers from surface to the sample point |
| ----- | ------------------------------------------- |
| Vp    | P velocity in kilometers per second         |
| Vs    | S velocity in kilometers per second         |
| Rho   | Density in grams per cubic centimeter       |
| Qp    | Attenuation for compressional waves         |
| Qs    | Attenuation for shear waves                 |

## Taup运行

### 编译.nd

进入.nd文件所在文件夹

```bash
taup create --nd 文件名.nd
```

Taup会输出一个同名的.taup文件

### 绘制震相

```bash
taup path --model 文件名.taup --deg 27 --evdepth 33 --phase PKPPKP,PKIKPPKIKP,PKIIKP,PKiKP,PKIKKIKP --svg -o mars_with_core.svg

```

- --deg：震中距离
- -- evdepth：震源深度
- --phase：震相
- --svg：输出格式为svg

"PKIKPKIP","PKKP","PKPPKP","PKIIKP","PKiKP","PKIKKIKP"

## 计算地震走时

### 无核走时

```bash
taup time --model mars_nocore.taup --deg 27 -h 33 --phase PKPPKP,PKKP --text -o travel_times.txt
```

### 有核走时

```bash
taup time --model mars.taup --deg 27 -h 33 --phase PKPPKP,PKIKPPKIKP,PKIIKP,PKiKP,PKIKKIKP --text -o travel_times.txt
```
